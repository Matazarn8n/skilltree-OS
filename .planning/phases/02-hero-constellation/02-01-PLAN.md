Créer `apps/web/lib/constellation/geometry.ts` — module **sans React, sans DOM**, fonctions pures sur les données du catalogue (importer `SECTORS`, `SKILLS`, `HUB` depuis `../catalog`).

Constantes (exportées) :
- `W_STEP = 360 / 7` (≈51.4286°)
- viewBox logique carré : `SIZE = 2000`, centre `C = 1000`
- rayons : `R_HUB = 120` (amas particules), `R_SECTOR = 310` (nœud hub de secteur), `R_BRANCH_START = 430`, `R_MAX = 920`

`computeWheelLayout()` retourne `{ size, hub, sectors, branches, nodes, edges, particles }` :
1. **Fans** : secteur d'`order` i (0..6) → angle centre `alpha_i = -90 + i * W_STEP` (degrés ; conversion `rad = deg * Math.PI / 180`, `x = C + r*cos(rad)`, `y = C + r*sin(rad)`). Nœud secteur à `(alpha_i, R_SECTOR)`, label au-dessus.
2. **Branches par fonction** : pour chaque secteur, grouper ses jobs `SKILLS.filter(j => j.sector === slug)` par `j.function` en **conservant l'ordre d'apparition** dans le catalogue (déterministe). Fi = nombre de fonctions du secteur (5,5,5,5,5,3,6 — total 34). Branche j (0..Fi-1) : angle `beta_ij = alpha_i - SPREAD/2 + (j + 0.5) * SPREAD / Fi` avec `SPREAD = W_STEP * 0.82` (marge inter-fans).
3. **Nœuds jobs le long de la branche** : Nj jobs sur la branche, nœud k (0..Nj-1) à rayon `r_k = R_BRANCH_START + k * (R_MAX - R_BRANCH_START) / Math.max(Nj - 1, 1)` ; si Nj == 1, placer à mi-chemin `(R_BRANCH_START + R_MAX)/2`. Décalage tangentiel alterné `±14` unités (k pair/impair) pour désengorger les labels.
4. **Arêtes** : centre→nœud secteur (7), nœud secteur→premier nœud de chaque branche (34), puis nœud k→k+1 le long de chaque branche. Chaque arête porte `colorVar` du secteur.
5. **Label de fonction** : position au rayon `dernier nœud + 45`, même angle que la branche, avec `jobCount`.
6. **Particules du hub** : `mulberry32(20260611)` (implémenter le PRNG dans le fichier, ~6 lignes) → tableau de `{x, y, r, opacity}` : 90 particules en distribution radiale dans `R_HUB` (rayon = R_HUB * sqrt(u) pour densité uniforme, plus dense au centre avec `u**1.4`). Exporter aussi une tranche `particles.slice(0, 24)` utilisable en LOWFX.

`computeSectorLayout(sectorSlug)` — vue zoom secteur (cf. `captures/map_zoom/*.png` : nœud secteur en bas-centre, fonctions en éventail au-dessus) :
- Nœud secteur à `(C, 1560)` ; fonctions en demi-éventail au-dessus : branche j → angle `gamma_j = -90 - ARC/2 + (j + 0.5) * ARC / Fi` avec `ARC = 150` degrés, mesuré depuis le nœud secteur.
- Jobs de la branche j échelonnés du rayon `380` à `1250` depuis le nœud secteur, même formule d'échelonnement que la roue ; label de fonction au bout de la branche.
- Retourner aussi `{ prev, next }` = slugs des secteurs adjacents par `order` (modulo 7) pour les chevrons.

Types : exporter `WheelLayout`, `SectorLayout`, `NodePos { job: Skill; x: number; y: number }`, etc. Toutes coordonnées dans [0, SIZE].

Créer `apps/web/lib/constellation/useLowFx.ts` — hook client :
